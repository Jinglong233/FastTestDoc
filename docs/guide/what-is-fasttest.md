# MokaTest 是什么

FastTest 是一个面向软件研发团队的质量管理与自动化测试平台，覆盖从需求到发布的完整测试流程。

## 核心能力

### 1. 团队与项目管理

- **团队**：顶层工作空间，支持多团队切换。
- **项目**：团队下的数据容器，需求/用例/BUG/场景都归属项目。
- **权限**：RBAC 角色权限模型，支持团队级和项目级授权。

### 2. 质量管理（QA）

- **需求池**：Epic/Story 层级管理、状态自由流转、关联用例与 BUG。
- **BUG 池**：完整状态机、严重程度/优先级、关联需求和用例。
- **用例列表**：按模块和测试集组织、支持自动化绑定、状态流转。
- **测试计划**：组合用例批量执行，记录执行结果和报告。
- **血缘追踪**：思维导图展示需求 → 用例 → BUG 关系。

### 3. 接口测试

- **接口管理**：目录树组织、Swagger 导入、接口用例保存。
- **接口调试**：HTTP 请求发送、断言、提取规则、前后置脚本。
- **Mock 响应**：字段规则/原始 JSON 两种模式，支持参数级 Mock。
- **环境/全局变量**：多环境配置、团队级共享变量。
- **数据模板**：可视化字段规则、批量生成、JSON/CSV/Excel 导出。

### 4. UI 自动化

- **元素库**：集中管理页面元素定位信息。
- **UI 场景**：可视化步骤编排，支持循环/if/断言等。
- **场景调试**：支持暂停/继续、单步调试、失败截图。

### 5. 测试运行与报告

- **任务计划**：组合 UI/API 场景，支持 Cron 定时和手动执行。
- **Webhook 通知**：支持钉钉/企业微信/飞书/自定义机器人。
- **测试报告**：汇总执行结果、查看请求响应、截图、视频、Trace。

## 适用场景

- 研发团队需要统一的需求、用例、BUG 管理平台。
- 测试团队希望构建可维护的接口/UI 自动化测试资产。
- 后端接口未就绪时，通过 Mock 提前完成前端和测试开发。
- 需要构造大量测试数据，复用数据模板生成规则。

## 技术栈

- 前端：Vue 3 + Vite + TypeScript + Arco Design
- 后端：Spring Boot + MyBatis-Plus + Sa-Token
- 数据库：MySQL 8.0+
- 文档：VitePress
