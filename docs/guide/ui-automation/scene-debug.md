# 场景调试

UI 场景编排完成后，可在编辑器内直接调试执行。

## 运行配置

执行前可配置运行参数：

| 参数 | 说明 |
|------|------|
| 浏览器类型 | Chrome / Edge / Firefox 等 |
| 窗口模式 | 有头/无头 |
| 超时时间 | 全局等待超时 |

::: tip 截图占位
此处需补充「场景运行配置弹窗」截图。
:::

## 调试执行

点击「调试」按钮后，后端启动浏览器执行场景步骤：

- UI 场景走 `/api/ws/debug` WebSocket 通道
- 支持暂停/继续操作
- 实时返回步骤执行状态

::: tip 截图占位
此处需补充「场景调试执行中」截图，展示步骤执行进度。
:::

## 单步调试

支持逐条执行步骤，便于定位问题。

## 执行结果

调试完成后可查看：

- 每个步骤的执行状态（成功/失败/跳过）
- 失败步骤的错误信息
- 页面截图（失败时自动截取）
- 执行日志

## 与 API 场景的区别

| 特性 | UI 场景 | API 场景 |
|------|---------|----------|
| 调试通道 | `/api/ws/debug` | `/api/ws/apiSceneDebug` |
| 暂停/继续 | 支持 | 不支持 |
| 主要资源 | 元素库 | 接口定义 |
| 输出产物 | 截图、视频 | 请求/响应详情 |
