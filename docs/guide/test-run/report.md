# 测试报告

测试报告汇总一次任务计划或 QA 测试计划的执行结果。

## 报告字段

| 字段 | 说明 |
|------|------|
| 报告名称 | 报告标题 |
| 任务类型 | UI / API / QA |
| 场景数 | 执行的场景总数 |
| 步骤数 | 执行步骤总数 |
| 断言数 | 断言总数 |
| 通过数 | 通过的场景/步骤/断言数 |
| 失败数 | 失败的场景/步骤/断言数 |
| 跳过数 | 跳过的场景/步骤/断言数 |
| 执行时长 | 总耗时 |
| 执行人 | 触发执行的用户 |
| 视频路径 | UI 执行录屏 |
| 场景详情 | 每个场景的详细结果（JSON） |

## 报告列表

报告列表页可按任务类型、状态、执行人等条件筛选。

::: tip 截图占位
此处需补充「测试报告列表页」截图。
:::

## 报告详情

报告详情页展示：

- 顶部统计卡片（总场景/通过/失败/跳过）
- 场景执行结果列表
- 每个步骤的请求/响应详情（API）
- 失败截图和错误信息（UI）
- 执行日志和 Trace

::: tip 截图占位
此处需补充「测试报告详情页」截图。
:::

## 视频与截图

UI 任务执行过程中会自动录制视频和截取关键画面，可在报告详情中回放。

## 相关接口

| 接口 | 说明 |
|------|------|
| `POST /report/reportPageList` | 报告列表 |
| `GET /report/allReport` | 所有报告 |
| `GET /report/reportDetail` | 报告详情 |
| `GET /report/deleteReport` | 删除报告 |
| `GET /screenshots/{filename}` | 读取截图 |
| `GET /automation/trace/{fileId}` | 下载 Trace 文件 |
