# 自动化测试

FastTest 的自动化测试能力覆盖接口测试、UI 自动化、测试计划与报告，帮助团队把单次手动执行沉淀为可反复运行的资产。

## 能力地图

| 能力 | 入口页面 | 说明 |
|------|---------|------|
| 接口测试 | `/interface-test/apiList` | HTTP 接口定义、调试、Mock、断言、数据模板 |
| API 场景 | `/interface-test/sceneList` | 把多个接口请求编排成可复用的自动化场景 |
| UI 自动化 | `/ui` | 元素库 + UI 场景编排 + 调试 |
| 测试计划 | `/plan` | 组合 UI/API 场景，支持手动 / 定时触发 |
| 测试报告 | `/report` | 汇总执行结果、请求响应、截图、Trace |
| 环境/全局变量 | `/interface-test/envConfig` | 团队级多环境配置和共享变量 |

## 典型工作流

1. **准备环境**：在环境管理中配置好测试环境的基础 URL、Header 和变量。
2. **定义接口 / 元素**：
   - 接口测试：录入 HTTP 接口，配置请求参数、断言、Mock。
   - UI 自动化：在元素库中维护页面元素定位信息。
3. **编排场景**：
   - API 场景：拖拽接口请求步骤，设置循环、条件、断言。
   - UI 场景：拖拽 UI 操作步骤，引用元素库。
4. **运行与报告**：把场景加入测试计划，手动执行或配置 Cron 定时执行，查看测试报告。
