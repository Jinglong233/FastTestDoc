# 项目管理

项目是团队下的数据容器，所有测试相关数据都归属于项目。

## 创建项目

进入团队工作台后，点击「新建项目」创建项目。项目需要填写名称、描述，并关联到当前团队。

::: tip 截图占位
此处需补充「团队工作台/项目列表」和「新建项目弹窗」截图。
:::

## 项目概览

进入项目后，项目概览页展示两类统计信息：

- **质量管理**：需求、BUG、用例等统计卡片。
- **自动化测试**：UI/API 场景数、任务计划、报告等统计。

::: tip 截图占位
此处需补充「项目概览页」截图。
:::

## 切换项目

在顶部导航栏的项目下拉框切换当前项目。切换项目时会重新加载当前用户在该项目下的权限。

::: tip 截图占位
此处需补充「顶部项目切换下拉框」截图。
:::

## 项目成员

除了团队级成员，每个项目还有独立的成员列表。项目成员从团队成员中选取，可分配项目级角色。

::: tip 截图占位
此处需补充「项目成员管理页」截图。
:::

## 相关接口

| 接口 | 说明 |
|------|------|
| `GET /project/allProject` | 获取项目列表 |
| `GET /project/getProjectById/{id}` | 项目详情 |
| `POST /project/addProject` | 创建项目 |
| `POST /project/updateProject` | 更新项目 |
| `POST /project/deleteProject/{projectId}` | 删除项目（逻辑删除） |
| `GET /project/member/list/{projectId}` | 项目成员列表 |
| `POST /project/member/assign` | 分配项目角色 |
| `POST /project/member/remove` | 移除项目成员 |
