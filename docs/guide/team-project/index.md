# 团队与项目

FastTest 采用 **团队 → 项目 → 数据** 三级组织方式。

## 组织关系

```
团队 A
├── 项目 A1
│   ├── 需求 / BUG / 用例
│   ├── 接口 / UI 场景
│   └── 测试计划 / 报告
├── 项目 A2
│   └── ...
└── 团队成员 / 角色模板 / 全局变量

团队 B
└── ...
```

## 数据隔离规则

| 层级 | 隔离内容 |
|------|---------|
| 团队 | 项目、成员、角色模板、环境、全局变量 |
| 项目 | 需求、BUG、用例、模块、场景、计划、报告 |

## 典型使用流程

1. 创建团队，邀请成员加入。
2. 在团队下创建项目。
3. 进入项目，开始录入需求、编写用例、执行自动化测试。
4. 通过角色和权限控制成员能看什么、能操作什么。
