# 环境与全局变量

环境和全局变量用于管理不同部署环境的基础配置和团队级共享变量。

## 环境（Environment）

环境归属于团队，可被团队下所有项目使用。

### 环境配置内容

| 配置项 | 说明 |
|--------|------|
| 环境名称 | 如：开发环境、测试环境、生产环境 |
| 基础 URL | 服务基础地址 |
| Header | 全局请求头 |
| Cookie | 全局 Cookie |
| 变量 | 环境变量键值对 |
| 数据库 | 数据库连接配置 |

::: tip 截图占位
此处需补充「环境配置弹窗」截图。
:::

### 使用场景

接口调试时选择环境，请求路径中的变量 `${baseUrl}` 会被替换为环境基础 URL。

## 全局变量（GlobalVar）

全局变量是团队级共享变量，分为四种类型：

| 类型 | 说明 |
|------|------|
| HEADER | 全局请求头 |
| COOKIE | 全局 Cookie |
| VARIABLE | 普通变量 |
| ASSERT | 全局断言 |

::: tip 截图占位
此处需补充「全局变量列表」截图。
:::

### 变量引用

在接口路径、参数、Body 中可通过 `${变量名}` 引用环境变量或全局变量。

## 权限控制

- 环境查看/编辑/删除需要 `auto:env:view/update/delete` 权限
- 全局变量查看/编辑/删除需要 `auto:globalvar:view/update/delete` 权限
