# 质量管理

质量管理模块覆盖需求、BUG、用例、测试计划和血缘追踪，是 FastTest 的核心模块之一。

## 模块组成

| 页面 | 路由 | 说明 |
|------|------|------|
| 需求池 | `/qa/requirement` | 管理产品需求，支持 Epic/Story 层级 |
| BUG 池 | `/qa/bug` | 跟踪缺陷全生命周期 |
| 用例列表 | `/qa/testCase` | 管理测试用例，支持绑定自动化 |
| 测试计划 | `/qa/testPlan` | 组合用例并执行 |
| 血缘追踪 | `/qa/traceability` | 查看需求 → 用例 → BUG 关联关系 |

## 数据组织

QA 数据以项目为边界，每个项目有独立的：

- QA 模块树（`qa_module`）
- 需求池（`requirement`）
- BUG 池（`bug`）
- 用例（`test_case`）
- 测试集（`test_case_set`）
- 测试计划（`test_plan`）

## 典型工作流程

1. 产品经理在需求池录入需求。
2. 测试人员围绕需求编写用例，将用例归入模块或测试集。
3. 用例执行发现问题后，在 BUG 池提 BUG 并关联需求和用例。
4. 通过测试计划批量执行用例，记录执行结果。
5. 通过血缘追踪查看需求、用例、BUG 之间的关联。
