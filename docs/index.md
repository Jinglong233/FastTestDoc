<div class="ft-section ft-section-alt">
  <h2 class="ft-section-title">自动化测试，两条路线都能走</h2>
  <div class="ft-auto-grid">
    <div class="ft-auto-card">
      <div class="ft-auto-header">
        <div class="ft-auto-icon icon-api">API</div>
        <h3>接口测试</h3>
      </div>
      <ul>
        <li>HTTP 接口请求、断言、提取、后置脚本</li>
        <li>参数级 Mock：后端没开发完也能调通场景</li>
        <li>数据模板批量生成请求数据和 Mock 响应</li>
        <li>环境变量 / 全局变量一键切换多套环境</li>
        <li>接口用例保存复用，接入测试计划批量执行</li>
      </ul>
    </div>
    <div class="ft-auto-card">
      <div class="ft-auto-header">
        <div class="ft-auto-icon icon-ui">UI</div>
        <h3>UI 自动化</h3>
      </div>
      <ul>
        <li>元素库集中管理，一处改动全局生效</li>
        <li>可视化场景步骤编排，拖拽组合测试流程</li>
        <li>UI 场景调试支持暂停 / 继续</li>
        <li>API 场景独立执行通道，和 UI 场景互不干扰</li>
        <li>场景接入测试计划，定时 / 手动触发执行</li>
      </ul>
    </div>
  </div>
</div>

<div class="ft-section">
  <h2 class="ft-section-title">为什么选 FastTest？</h2>
  <div class="ft-cards">
    <div class="ft-card">
      <div class="ft-card-tag tag-qa">QA</div>
      <h3>需求不是孤立的</h3>
      <p>Epic / Story 层级管理，需求改动时自动提示关联用例需要回归。</p>
    </div>
    <div class="ft-card">
      <div class="ft-card-tag tag-bug">BUG</div>
      <h3>BUG 从哪来，到哪去</h3>
      <p>每个 BUG 都能追溯到需求和用例，关闭原因必须填写，拒绝"莫名消失"。</p>
    </div>
    <div class="ft-card">
      <div class="ft-card-tag tag-case">CASE</div>
      <h3>用例是资产，不是一次性文件</h3>
      <p>文件夹分类、版本记录、自动化绑定，让用例越积越值钱。</p>
    </div>
    <div class="ft-card">
      <div class="ft-card-tag tag-perm">PERM</div>
      <h3>权限按团队 + 项目管控</h3>
      <p>团队、项目、模块三级权限隔离，不同项目看到的数据互不串门。</p>
    </div>
    <div class="ft-card">
      <div class="ft-card-tag tag-safe">SAFE</div>
      <h3>误删了能找回</h3>
      <p>核心模块支持逻辑删除，关联数据按需解绑，不用怕手滑把资产清掉。</p>
    </div>
    <div class="ft-card">
      <div class="ft-card-tag tag-trace">TRACE</div>
      <h3>血缘追踪一张图</h3>
      <p>需求 → 用例 → BUG 的关联关系用思维导图展开，问题定位更快。</p>
    </div>
  </div>
</div>
